Objektum orientált paradigma és annak megvalósítása a JAVA és C++ nyelvekben.
Az absztrakt adattípus, az osztály. Az egységbe zárás, az információ elrejtés, az
öröklődés, az újrafelhasználás és a polimorfizmus. A polimorfizmus feloldásának
módszere.

Absztrakt adattipus
	-absztrakt osztaly azt jelenti, hogy nem lehet belole objektumot csinalni csak leszarmaztatni
	-Pl.: Allat osztaly es hangot_ad() fuggveny, allat nincs ,,altalanos"  allat hang
		  Viszont kutyanak van, ezert csak leszarmaztathato az allat osztaly es a fuggveny a leszarmazott osztalyban van csak megvalostiva
	-c++ nincs abstract kulcsszo, ezt egy pure virtual fuggvennyel lehet megvalositani
	
Kulonbseg Java es c++ kozott
	-Oroklodes: Java egyszeres de tobb interfeszt is implementalhat , C++ gyemant
		(interfesz = blueprint(metodusok))
	-Memoria: Java garbage collector, manualis
	-Interfesz: Java van, C++ nincs
	-Polimorfizmus: Java automata, C++ virtual kulcsszo
	-Absztrakt: Java abstract kulcsszo, C++ pure virtual (virtual a fuggveny ele es =0)
	-Futas: Java JVM-en fut, C++ processzoron fut -> gyorsabb
